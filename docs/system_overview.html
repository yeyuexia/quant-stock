The big picture

What the system does, in one breath

Every trading day the system asks three questions and answers each with the right tool:

What should I buy?
Two independent strategies each rank stocks (one on cheapness+quality, one on growth+momentum). A single AI agent reads both lists and picks the top 4. Slow, judgment-heavy → runs once a day.
When do I buy it?
The watchdog watches those 4 names intraday and fires a buy only when a coded volume-breakout rule confirms the move. Fast, deterministic → runs every 5 minutes.
How do I not blow up?
Every order passes a safety layer (kill-switch, cash check, daily caps, large-order gate). Positions get stop-losses and rule-based exits. Nothing trades on real money — it's a paper account.
Design principle that runs through everything: separate the slow, smart, daily decision (which stocks — done by the LLM) from the fast, deterministic, high-frequency decision (when to buy / when to exit — done by code). The LLM never sits on the intraday hot loop.
End-to-end flow

The five stages

Data flows left to right. Each stage has one job and hands off to the next through a file or a function call.

1 · Signals

Momentum, CANSLIM screen + VCP bases, value/quality, macro regime.

momentum.py screener.py value_screen.py macro.py

2 · Selection

Strategies write ranked lists; the AI agent picks the top 4 buy candidates.

strategies.py investor_agent.py

3 · Timing

Watchdog monitors candidates & holdings; coded rules decide buy / exit moments.

watchdog.py sepa_exits.py

4 · Execution

Targets become sliced orders; circuit breakers gate intraday submission.

rebalancer.py planner.py executor.py

5 · Safety & Broker

Every order funnels through the policy layer, then to Alpaca.

orders.py broker.py → Alpaca
→→→→

Invariant: every order — from a rebalance, a buy signal, a stop, or a SEPA exit — passes through orders.py, which enforces HALT / paper-live guard / daily caps / large-order approval. broker.py is pure I/O with no policy.

The AI ensemble (newest layer)

One Claude agent

Merges both lists, dedupes (consensus boosts a name), drops owned/illiquid, then asks the claude CLI for the top 4 + rationale. If the LLM fails, it falls back to a deterministic rule ranking.

Buy candidates

.cache/buy_candidates.json — the only names the watchdog will consider buying that day. Read-only to the watchdog; a staleness guard ignores it if the daily job hasn't refreshed it.
Why it runs once a day, not every 5 minutes: the agent's only inputs are the daily screens — re-running the LLM intraday can't produce a better answer (no new information), just more cost and churn. So selection happens once, in the pre-market watchdog run; the intraday loop only does coded timing.
A day in the life (local time; US-Eastern market hours)

What runs, and when

Scheduled by cron. Times below are the machine's local time, mapped to US market hours.

20:30 · Daily watchdog= 8:30 AM ET, pre-market

Syncs the live portfolio, attaches missing stops, checks macro — and runs the AI ensemble once (strategies in parallel → agent → buy_candidates.json). Fail-open: a generation hiccup never breaks the watchdog.

21:35 · Rebalancer= 9:35 AM ET

Builds two-tranche target weights (core ETFs + stocks, aggressive leveraged ETFs), writes the plan to pending_plan.json. Large orders queue for approval; tiny ones submit directly.

22:00–04:00 · Intraday loopevery 5–10 min, RTH only

Executor works the pending plan through 5 circuit breakers. Watchdog --intraday reads the buy candidates and fires coded buy signals, enforces stop-losses (market-sell on breach, works on fractional shares), and runs SEPA exits. No LLM here.

07:00 · Quant review= post-close, Tue–Sat

A separate Claude review writes strategy overrides + a Telegram note for the next day's rebalance.

Sun 20:00 · Discoveryweekly

Refreshes the auto-discovered watchlist (S&P 500 + Nasdaq-100 + S&P 400) before Monday's rebalance.

Under the hood

Components — click any to expand

Signals Selection Timing Execution Safety / data
screener.pysignals
CANSLIM technical screen: quarterly EPS/revenue growth pre-filter, relative-strength percentile, and VCP (Volatility Contraction Pattern) base detection with a buy pivot. One of the two ensemble strategies.
value_screen.pysignals
Value + quality screen. Hard liquidity/price/market-cap/anti-trap gates, then a winsorized cross-sectional z-score blending value (FCF yield, earnings yield, EV/EBITDA, book/market), quality (ROE, low debt), and improving fundamentals. The "low value, high potential" source.
momentum.py / macro.pysignals
Dual-momentum ETF rotation for the core sleeve, and a FRED-based macro regime overlay that shrinks risk in hostile regimes and can trigger macro-driven exits.
strategies.pyselection
The isolated-strategy contract + parallel runner. Each strategy writes {ticker, score, rank, factors} rows to its own JSON file; a strategy that errors or hangs is skipped (bounded by a timeout) so one can never stall the others or the watchdog.
investor_agent.pyselection
The single Claude agent. select_candidates() merges the strategy lists, dedupes with consensus weighting, excludes owned names, calls the claude CLI for the top-N picks with rationale, validates the JSON, and falls back to rule-ranking on any LLM failure.
watchdog.pytiming
The daily + intraday monitor. Daily run: sync, stop attach, macro, and the once-a-day ensemble trigger. Intraday run (every 5 min, RTH): coded buy signals from the candidate list, stop-loss enforcement (auto market-sell on breach), and SEPA exits. The candidate source prefers buy_candidates.json and falls back to the raw screener.
sepa_exits.pytiming
Pure Minervini SEPA exit rules: R-multiple scale-outs, failed-breakout detection, climax-top detection, and a 21-EMA trailing backstop. The watchdog calls into it.
rebalancer.pyexecution
Builds target weights per tranche from the signals, sends the tiny actionable slice through orders.py, and writes the rest to pending_plan.json for the intraday executor.
executor.pyexecution
Intraday driver (every 10 min, RTH). Consumes the pending plan, evaluates 5 circuit breakers (SPY drop, VIX spike, single-name shock, news shock, macro flip), and submits sliced marketable-limit orders.
planner.py / planning.pyexecution
Pure functions that enrich raw order intents with execution tier, price limits, and slice counts. No I/O.
orders.pypolicy
The policy chokepoint. sync_state reconciles live positions and adopts broker-imported holdings into a sleeve; execute_plan applies all safety rails; pending-queue + stop helpers live here. Every order goes through it.
broker.pydata
Thin Alpaca SDK wrapper — pure I/O, raises BrokerError, leaks no SDK objects. The paper/live guard requires two explicit env flags to ever touch a live account.
Why it doesn't blow up

Risk & safety rails

Every order — no matter the trigger — passes these in orders.py:

HALT kill-switch

Drop a .cache/HALT file and all order logic exits cleanly. touch to pause, rm to resume.

Paper / live guard

Live trading needs both ALPACA_ENV=live and an explicit confirm flag. Any single-flag typo stays on paper.

Cash-aware gate

Buys can't exceed available cash (unless margin is explicitly allowed); a broker-fetch failure fails closed.

Daily caps

Hard limits on order count and total notional per day; excess defers to tomorrow.

Large-order approval

Orders above a threshold queue to pending_orders.json for Telegram approval instead of firing.

Position adoption + starvation alert

Broker-imported "unknown" positions are auto-tagged into a sleeve so the rebalancer manages them; a loud alert fires if untagged value would starve it.

Fractional-safe stops

Native stops can't attach to fractional shares, so the watchdog enforces stops by market-selling on breach — anchored to each position's adoption date so a freshly-adopted name isn't liquidated against an old high.

Candidate staleness guard

If the daily ensemble job hasn't refreshed buy_candidates.json within a max age, the watchdog ignores it and falls back — so auto-buys never run on stale picks.

Getting out

Exit logic

Selling is rule-driven, never discretionary. The core stock sleeve uses Minervini SEPA; everything has a stop.

Stop-loss

Per-sleeve stop (ATR-scaled for core, fixed for aggressive). Breach → market sell, enforced intraday.

Trailing stop

Peak-to-current trail, measured from when the position was first managed (not its full history).

SEPA R-multiple scale-out

Sell a fraction at 2R and 3R profit tiers, banking gains as the move extends.

Failed breakout

If a fresh breakout reverses below its pivot within a few days, exit — the thesis failed.

Climax top

Parabolic blow-off detection (extended run + range/volume expansion) tightens the trail.

21-EMA backstop

After the final scale-out tier, exit the remainder on a daily close below the 21-EMA.

Macro flip

A hostile macro-regime change can drive defensive exits the next day.

Circuit breakers

Intraday SPY drop / VIX spike / single-name / news / macro shocks halt or abort in-flight execution.

Plain-English terms

Glossary

Tranche
A sub-portfolio with its own rules. Core (~90%) = diversified ETFs + screened stocks; Aggressive (~10%) = leveraged ETF momentum.
CANSLIM / VCP
A growth-stock methodology (O'Neil) and the tight "base" pattern that precedes a breakout — the buy pivot.
SEPA
Minervini's Specific Entry Point Analysis — the rule set for scaling out of winners and cutting failures.
R-multiple
Profit measured in units of initial risk. "2R" = twice the per-share risk taken at entry.
Ensemble
Running multiple independent strategies and combining their outputs — here, via the AI agent's top-N pick.
Buy candidates
The agent's daily shortlist (top 4). The only names the watchdog will consider buying that day.
Circuit breaker
An automatic trading halt/abort triggered by a market or single-name shock during execution.
Paper account
A simulated brokerage account — real prices, fake money. This system trades paper by default.
